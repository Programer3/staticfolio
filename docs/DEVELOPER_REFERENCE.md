# Developer Reference — staticfolio

> For developers who need to understand, debug, modify, or extend this portfolio template.

---

## 1. Overview

| Field | Value |
|-------|-------|
| **Project** | staticfolio |
| **Version** | 1.0.0 |
| **Tagline** | Dark-mode-first, brutalist/glassmorphism developer portfolio |
| **Tech Stack** | Astro 7, TypeScript, GSAP 3.15, Tailwind CSS v4 |

A single-page developer portfolio with 10 themes, live session metrics, custom cursor, micro-interactions, and zero-backend contact form. Statically exported for GitHub Pages — no server required.

### Key Files

| File | Purpose |
|------|---------|
| `src/data/portfolio.ts` | All personal content (name, bio, skills, projects, socials) |
| `src/data/themes.ts` | 10 theme palette definitions |
| `src/styles/global.css` | Tailwind + theme tokens + utility classes + component styles |
| `src/layouts/Base.astro` | Document shell, meta tags, anti-FOUC script, JSON-LD |
| `src/pages/index.astro` | Single page composing all sections |
| `src/scripts/interactions.ts` | GSAP ScrollTrigger, magnetic buttons, parallax, Konami code |
| `src/scripts/countUp.ts` | Shared count-up animation utility |

## 2. Architecture

```
Base.astro (layout shell)
  └─ index.astro (page composition)
       ├─ Nav.astro (sticky glass nav + ThemeSwitcher + GooglyEyes)
       ├─ Hero.astro (value prop, parallax orbs, scroll cue)
       ├─ About.astro (bio + philosophy sidebar + LocCounter)
       ├─ Skills.astro (tool wall grid of SkillCards)
       ├─ Projects.astro (bento grid of ProjectCards)
       ├─ Contact.astro (form + direct channels)
       ├─ Footer.astro (links, credits, konami hint)
       └─ ActivityHUD.astro (floating session metrics widget)
```

### Data Flow

```
portfolio.ts ──┐
themes.ts   ──┤
               ├──→ Components (via import)
               ├──→ Base.astro (meta, JSON-LD)
               └──→ ThemeSwitcher (dropdown + localStorage persistence)
```

### Component Rendering

- All `.astro` components render to static HTML at build time.
- Client-side JavaScript scoped per component via `<script>` tags.
- GSAP and ScrollTrigger bundled into a single shared chunk via `interactions.ts`.
- Theme system: `data-theme` attribute on `<html>`, persisted in `localStorage`, applied synchronously before paint via inline script.

## 4. Module Walkthrough

### 4.1 `src/data/portfolio.ts`

Single source of truth for all personal content. Exports a `Portfolio` interface and a `portfolio` object. The `// ← edit` markers indicate the fields most users will change.

**Interfaces:**

| Interface | Fields | Purpose |
|-----------|--------|---------|
| `Skill` | `name`, `icon`, `use` | A tool/technology the developer uses |
| `Project` | `title`, `tagline`, `description`, `year`, `stack`, `live?`, `repo?`, `motif`, `accent`, `featured?` | A project to showcase |
| `SocialLink` | `label`, `href`, `icon` | A social media link |
| `Portfolio` | All top-level fields | The complete portfolio config |

**`motif` values** control the SVG preview: `orbit`, `grid`, `wave`, `circuit`, `prism`, `glyph`, `stack`, `pulse`.

### 4.2 `src/data/themes.ts`

10 theme definitions, each with 13 CSS token values. The `defaultThemeId` is `'panda'`.

**ThemeTokens:**

| Token | Description |
|-------|-------------|
| `bg` | Page background |
| `bgAlt` | Elevated surface |
| `surface` | Card / glass |
| `surfaceMuted` | Recessed chip |
| `border` | Hairline |
| `borderStrong` | Visible separator |
| `text` | Primary text |
| `textMuted` | Secondary |
| `accent` | Primary accent |
| `accent2` | Secondary accent |
| `accentText` | Text on accent |
| `mode` | `'dark'` or `'light'` |
| `font` | `'mono'` or `'sans'` |
| `grain` | Opacity of grain overlay (0–1) |
| `glow` | Accent shadow intensity (0–1) |
| `vibe` | Human-readable label |

### 4.3 `src/styles/global.css`

Contains:
- Tailwind CSS v4 import
- Fontsource font imports (Space Grotesk, JetBrains Mono, IBM Plex Mono)
- `@theme` block bridging CSS variables to Tailwind utilities
- All `[data-theme="..."]` variable blocks (10 themes + `:root` default)
- `.site-grid` and `.site-ambient` background layers
- Base layer: resets, typography, scrollbar, `::selection`, `:focus-visible`
- Utility classes: `.glass`, `.glass-strong`, `.gradient-text`, `.grid-bg`, `.magnetic`
- Modal primitives: `.modal-overlay`, `.skill-modal`, `.snake-modal`
- Magnetic button system: `.mb.*`, `.mb--primary`, `.mb--outline`, `.mb--ghost`
- Animation keyframes: `caret-blink`, `float-y`, `modal-in`, `modal-pop`, `mb-punch`, `mb-ripple`

### 4.4 `src/layouts/Base.astro`

Document shell that:
- Inlines a synchronous anti-FOUC theme script (reads `localStorage` before paint)
- Sets Open Graph, Twitter Card, and standard meta tags
- Injects JSON-LD structured data (Person schema)
- Renders background layers (grid + ambient orbs), GrainOverlay, Cursor
- Provides `<slot />` for page content
- Includes a console banner easter egg

### 4.5 `src/scripts/interactions.ts`

Global interactions imported once on the index page.

| Function | Purpose |
|----------|---------|
| `initReveals()` | GSAP ScrollTrigger fade-up reveals for `[data-reveal]` elements; staggered `[data-reveal-group]` children |
| `initParallax()` | Parallax layers via `[data-parallax]` speed factor |
| `initMagnetic()` | Magnetic pull on `[data-magnetic]` buttons (inner + outer elements) |
| `initKonami()` | Konami code listener → launches canvas snake game in a modal |

All functions degrade gracefully under `prefers-reduced-motion: reduce`.

### 4.6 `src/scripts/countUp.ts`

Shared count-up animation utility. Triggered by `IntersectionObserver` when elements scroll into view. Supports two easing curves:
- `easeOutCubic` (default) — smooth deceleration, used by LOC counter
- `easeExpoBack` — dramatic expo start → overshooting settle, used by hero experience counter

Controlled via data attributes: `data-count-up`, `data-target`, `data-duration`, `data-ease`, `data-visibility`.

## 5. Theme System Reference

### How Themes Work

1. `Base.astro` inlines a `<script>` that reads `localStorage.getItem('folio-theme')` and sets `data-theme` on `<html>` **before paint** — no flash of wrong theme.
2. `ThemeSwitcher.astro` renders a dropdown of all 10 themes. Selecting one updates `data-theme` and persists to `localStorage`.
3. `global.css` resolves all `--t-*` CSS custom properties based on `[data-theme="..."]`.
4. Components consume tokens via `var(--t-*)` or Tailwind utilities like `bg-t-surface text-t-text`.

### Adding a Theme

1. Add a new entry to the `themes` array in `src/data/themes.ts` with all 13 tokens.
2. Add a `[data-theme='your-id']` block in `src/styles/global.css`.
3. The theme appears automatically in the switcher dropdown.

## 7. Common Pitfalls

### Theme System
- **FOUC on slow connections** — The inline script handles this, but check if `localStorage` is blocked by browser settings.
- **Adding a theme but forgetting the CSS block** — The switcher will show it, but CSS variables won't resolve. Always add both `themes.ts` entry and `global.css` block.

### Activity HUD
- **Bars not animating** — The velocity sparkline only updates on scroll events. If you're on a page with no scroll, the bars stay at minimum height. This is expected.
- **HUD not showing** — Hidden on screens < 768px. Check viewport width.
- **Dragging broken** — Only works on desktop (non-touch). The `touch` variable check prevents drag setup on mobile.

### Custom Cursor
- **Cursor visible on mobile** — Should auto-hide. If not, check media query: `@media (hover: none), (pointer: coarse)`. The dot might briefly appear before JS kicks in.
- **Ring not growing** — Ensure targets have `[data-cursor-label]` or are `<a>/<button>/<summary>`/`<input>`/`<textarea>` elements.

### Contact Form
- **`mailto:` not opening** — Some browsers block `mailto:` from `fetch` contexts. It opens via `window.location.href` which should work, but may need user gesture. The form submit event provides this.
- **Formspree not receiving** — Ensure your endpoint URL is correct in `contactEndpoint`. Some providers require the form to have `method="POST"` and `action` set.

### Performance
- **Large OG image** — `og-image.svg` is an SVG and should be small, but if replaced with a PNG, keep it under 200KB to avoid Lighthouse warnings.
- **GSAP bundle** — At ~40KB gzipped, GSAP is the largest dependency. Consider removing `ScrollTrigger` if you don't need scroll reveals: remove `gsap/ScrollTrigger` import from `interactions.ts` and delete the `initReveals()` and `initParallax()` calls.

## 10. Testing

No automated test suite exists for this project. Testing is manual:

- **Theme switching** — Verify all 10 themes render correctly, persist across page reloads, and have no FOUC.
- **Responsive** — Check at 375px, 768px, 1024px, 1440px widths.
- **Reduced motion** — Enable `prefers-reduced-motion: reduce` in devtools. Verify all animations are disabled and content is visible.
- **Touch devices** — Verify cursor hides, tilt effects don't apply, HUD hides on mobile.
- **Keyboard nav** — Tab through all interactive elements, verify visible focus rings.
- **Contact form** — Submit with empty fields (should show validation errors), valid fields (should open mailto or POST), and test with JS disabled (should show noscript fallback).
- **Konami code** — Type `↑↑↓↓←→←→ba` and verify snake game loads. Press Escape to close.

## 11. Debugging

### Useful DevTools

- **Theme debugging**: `document.documentElement.dataset.theme` shows current theme. `getComputedStyle(document.documentElement).getPropertyValue('--t-accent')` reads any token.
- **Activity HUD**: `document.querySelector('[data-hud]')` to inspect. Check the `state` object in the HUD's script via breakpoints.
- **GSAP**: `gsap.utils.toArray('[data-reveal]')` in console to verify elements are targeted. Check ScrollTrigger for `start`/`end` markers.
- **localStorage**: `localStorage.getItem('folio-theme')` to check persisted theme.

### Common Debugging Scenarios

| Issue | Investigation |
|-------|--------------|
| Theme reverts on reload | Check localStorage write in ThemeSwitcher script. Check for CSP blocking inline scripts. |
| Scramble animation not working | Verify `prefers-reduced-motion` is not enabled. Check if on touch device. |
| Contact form not sending | Check `contactEndpoint` value. Look for console errors. Check network tab for POST. |
| GSAP reveals not firing | Ensure `interactions.ts` is imported in `index.astro`. Check ScrollTrigger is registered. |
| LOC counter shows "awaiting sync" | The GitHub Actions workflow hasn't run yet. Push to `main` or run the workflow manually. |
| Build errors | Check Node.js version (requires Node 18+). Verify all imports resolve. Clear `.astro/` cache. |
